Why XNo Websites Are Self-Hosted

XNo is built with a core philosophy: websites should be fast, reliable, and simple to maintain. To achieve this, XNo websites are self-hosted rather than relying on database-driven backends.

This means no external databases or servers are required to run your site — everything lives directly in your files and browser.

Each XNo site is hosted on a fork of the main repository that the user creates on GitHub, which is then automatically rehosted on platforms like Vercel for easy and fast global access. This seamless Git + Vercel workflow makes deployment and updates simple and serverless.

Benefits of a Self-Hosted Approach

  • Avoid Database Issues: No database means no risk of downtime or data loss caused by database failures or misconfigurations.
  • Fully Serverless: Your site runs entirely in the client’s browser, eliminating the need for backend servers or server maintenance.
  • Better Performance: Static hosting with smart client-side logic means faster load times and instant scaling.
  • Privacy & Control: You own your data completely. No centralized storage, no third-party dependencies.

XNo Plugins — Extending Your Site

Even though XNo sites are serverless and self-hosted, they are fully extensible through XNo plugins. These lightweight JavaScript modules add extra features and functionality without needing any backend infrastructure.

Plugins can be created by anyone — individuals, companies, or open-source communities — empowering a diverse ecosystem of enhancements.

  • Modular: Add only the plugins you need.
  • Easy to use: Install and configure with minimal setup.
  • Serverless by design: All plugins run client-side, keeping your site simple and reliable.

Explore available plugins at xnocommunity.vercel.app/plugins.

To get started, fork the main repository at https://github.com/electric-otter/xno/fork and customize your site or plugins from there.

If you don’t want to fork, you can also submit a pull request to add your site as a folder domain, like yourxnosite.xno, in the main repository.
